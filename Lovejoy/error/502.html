HTTP 502 - Bad Gateway No Response

Error Explanation: Bad gateway, HTTP request to backend node, no response.
Reason 1: DNS Cache
Solution:
Run the command 'ipconfig /flushdns' in a DOS window to refresh the DNS cache.
Reason 2: Browser Proxy
Solution:
Turn off the proxy.
Reason 3: DNS hijacking, even using foreign DNS can be hijacked
Solution:
Remove the DNS from the VPN server. Switch to another DNS. In Windows systems, you can change the properties of the local network connection to use a foreign DNS, such as Google's or OpenDNS.
Reason 4: PHP execution timeout
Solution:
Modify '/usr/local/php/etc/php.ini' and change 'max_execution_time' to 300.
Reason 5: nginx wait time timeout
Solution:
Increase the FastCGI timeout time in the 'nginx.conf' configuration file appropriately.
